# Claude Setup Checklist

## Goal
Get Claude Code operating on this vault without letting it freewheel over structure, names, or sensitive material.

## Before first session
- keep the vault in a repo if you want version history
- place `CLAUDE.md` at the root
- make sure root docs exist and are readable
- decide whether hidden material exists only as shadow references or also in an external encrypted lane

## First command sequence inside Claude Code
1. ask Claude to read `README.md`
2. ask Claude to read `VAULT_RULES.md`
3. ask Claude to read `AI_USAGE.md`
4. ask Claude to read `MODE_AND_ACCESS_MODEL.md`
5. ask Claude to summarize the allowed and disallowed behaviors back to you

## First safe tasks
- validate frontmatter
- build manifest
- extract evidence units from a chosen folder
- update one profile note from linked evidence
- create one export packet

## Tasks to avoid first
- mass renaming
- broad tag changes
- plugin or automation installation without review
- hidden-lane handling beyond shadow references
