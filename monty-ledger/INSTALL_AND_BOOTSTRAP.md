# Install and Bootstrap

## Objective

Get the vault operational fast without turning setup into its own hobby.

## Step 1. Create or copy the vault

Place this folder where you keep your main notes.
Open it in Obsidian as a vault.

## Step 2. Read the root files

Read in this order:
1. README
2. VAULT_RULES
3. AI_USAGE
4. MODE_AND_ACCESS_MODEL
5. PARA_AND_OVERLAY

## Step 3. Obsidian settings

Enable or confirm:
- Properties
- Templates
- Daily notes
- Canvas
- Backlinks
- Command palette
- Quick switcher

Review `.obsidian/` files in this package.
Adjust only if something clearly gets in your way.

## Step 4. Community plugins

Suggested first set:
- Dataview
- Templater
- Tasks
- Calendar
- one visual tool only

Do not install ten more because they look cool.

## Step 5. First notes to make real

Replace placeholders or expand:
- `Profile - Identity and Context`
- `Profile - Voice and Writing`
- `Profile - Roles and Modes`
- `Profile - Values and Duty`
- `Project - Vault Implementation`
- one current live project
- three current decisions

## Step 6. First evidence import

Bring in 10 strong evidence units from recent chats or notes.
Link them to profile notes.

## Step 7. Claude Code setup

If using Claude Code:
- place the vault in a repo if you want version control
- read `CLAUDE.md`
- review `15_Agent_Prompts/`
- use `/config` as needed inside Claude Code
- have Claude read the root rules before making structural edits

## Step 8. First export packet

Create one packet for one real task under `exports/`.
Do not dump the whole vault into an external model.

## First-month rule

No redesign for the first month unless something is clearly broken.
Use it first.
