# PARA and Overlay

This vault is not anti-PARA.
It uses PARA where PARA is strong and adds what PARA does not model well enough.

## What PARA is strong at

PARA is strong for broad digital organization:
- Projects
- Areas
- Resources
- Archive

That makes it practical and fast for action and retrieval.

## Where PARA is weak for this system

PARA alone does not model:
- layered identity
- access control
- truth layers
- masks and roles
- quote provenance
- contradiction tracking
- voice rules
- evidence-backed profile building

## This vault uses a dual model

### PARA-like retrieval layer

- `02_Projects` maps to Projects
- ongoing responsibilities sit across `03_Profiles`, `05_Leadership`, `06_Writing`, `08_Knowledge`
- reference material sits in `08_Knowledge` and `09_Assets`
- inactive material goes to `99_Archive`

### Identity and reasoning overlay

- `03_Profiles`
- `04_Decisions`
- `05_Leadership`
- `06_Writing`
- `07_Life_Arcs`
- `13_Pods`
- `14_Lenses`

These are the layers BASB/PARA usually do not build out enough for the kind of system this vault is trying to be.

## Working principle

Use PARA for:
- action
- storage
- broad retrieval

Use the overlay for:
- self-modeling
- mode control
- evidence-backed reasoning
- AI-ready context loading

## Bottom line

PARA is a good floor.
It is not the full building.
