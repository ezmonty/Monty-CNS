# Team Audit Plan

## Structure

Three-member review board.

### Reviewer 1. Structure and Retrieval Lead
Looks at:
- folder logic
- metadata quality
- dashboard usefulness
- retrieval speed

### Reviewer 2. Voice and Identity Integrity Lead
Looks at:
- voice fit
- raw versus output preservation
- quote provenance
- whether the system sounds like itself

### Reviewer 3. Access and Risk Lead
Looks at:
- public/private/secret/hidden handling
- role modes
- mask levels
- leak control
- what should not load by default

## Compilation method

Each reviewer produces:
- strengths
- holes
- required changes
- optional changes
- rubric score

Then the lead compiles:
- overlaps
- conflicts
- tie-break items
- final score
- release decision

## Tie-break rule

If a change is strategic rather than obviously structural, it goes to the user.
