# AI Usage

## Main idea

This vault is meant to be read by AI through:
- markdown structure
- frontmatter metadata
- wikilinks
- packet exports
- profile notes
- evidence units
- decision records

AI should answer from the recorded system first, not from generic instincts first.

## Retrieval order for drafting

1. relevant project note
2. relevant profile notes
3. relevant decision notes
4. strongest evidence units
5. current draft target

## Retrieval order for strategy

1. `Profile - Identity and Context`
2. `Profile - Values and Duty`
3. `Profile - Reasoning and Decision`
4. `Profile - Work and Execution`
5. related decisions
6. related project note

## Retrieval order for personal reflection

1. `Profile - Life Arc`
2. `Profile - Values and Duty`
3. `Profile - Voice and Writing`
4. `Profile - Leadership`
5. evidence notes tied to the topic

## AI must

- read metadata first
- note confidence and uncertainty
- preserve quote provenance
- distinguish raw from output layers
- avoid flattening tensions into clean summaries
- prefer canonical notes over scattered comments

## AI must not

- invent durable traits from one moment
- erase contradictions that might matter
- rewrite internal notes for polish unless asked
- load sensitive material into unrelated tasks
- assume the most polished text is the deepest truth
- assume the rawest text is the wisest truth

## External model use

When using NotebookLM, ChatGPT, Claude, or another system:
- export a packet for one question at a time
- include the packet index
- include the profile notes that actually matter
- keep the packet tight

One question, one packet.
