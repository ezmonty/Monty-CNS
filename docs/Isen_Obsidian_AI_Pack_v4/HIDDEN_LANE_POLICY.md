# Hidden Lane Policy

## Decision
Truly hidden material lives outside the main vault.

The main vault may know that something exists. It may not automatically store the thing itself.

## Why
Some material is too volatile, intimate, compromising, or causally charged to sit inside normal operational memory.
That does not mean it has no influence. It means the influence should be modeled without casual access to the source.

## Main-vault rule
The main vault stores only:
- a shadow identifier
- a safe abstract summary
- influence domains
- retrieval limits
- exposure rules

The main vault does not store the full underlying narrative by default.

## Three storage levels

### Level 0
Do not store the event at all.
Only store current behavior rules if needed.

### Level 1
Store a shadow reference only.
This is the default.

### Level 2
Store a brief external encrypted note if there is a clear reason.
Examples:
- therapy support
- pattern tracking
- motive analysis
- contradiction review

### Level 3
Store a fuller narrative only if there is a specific therapeutic, legal, or practical reason to do so.

## Rule of burden
No hidden note gets written just because systems people say it should.
The note has to earn its keep.

## Retrieval rule
Hidden-lane material only loads when one of these is true:
- the task directly concerns healing, therapy, trauma, or pattern work
- the note explains a recurring distortion relevant to the current issue
- the user explicitly asks to include hidden-layer factors

## Security stance
If hidden material exists, prefer encrypted storage outside the main vault.
The main vault should carry only the shadow reference and influence note.
