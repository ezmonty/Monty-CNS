# Profile - Work and Execution

## Operating style

The preference is for usable structure, not decorative structure.

## Work support that helps

- clear next actions
- real tradeoffs
- visible consequences
- tools that reduce retrieval friction
- systems that survive contact with real workloads

## Work support that does not help

- productivity theater
- too many knobs
- frameworks that assume unlimited time to maintain themselves
- advice that ignores money, delivery, trust, or operational drag

## AI rule

Outputs should be:
- actionable
- grounded
- aware of cost
- aware of sequence
- honest about what is missing
