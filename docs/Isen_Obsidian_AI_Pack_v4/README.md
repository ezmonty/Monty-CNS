# Isen OS Vault

This vault is not a note pile. It is a recorded reasoning field.

The point is simple:

- store what changes future output
- separate raw thought from approved output
- preserve quote provenance
- preserve role, access, and mask differences
- make the vault usable by humans first and AI second, without turning it into decoration

## What this vault is for

This vault is meant to hold:

- ongoing projects
- profile layers
- leadership thinking
- voice and revision rules
- decision records
- life arc notes
- evidence units
- AI-ready export packets

It is designed so that a future model or agent can answer from recorded standards, recorded decisions, and recorded patterns instead of starting cold.

## What this vault is not for

- vague self-improvement journaling
- infinite dashboard building
- saving everything just because it happened
- turning every note into polished prose
- pretending there is one flat self that always speaks the same way

## Folder map

- `00_Inbox` fast capture only
- `01_Daily` daily notes and logs
- `02_Projects` active projects
- `03_Profiles` canonical identity notes
- `04_Decisions` decision logs
- `05_Leadership` leadership thinking
- `06_Writing` voice, draft, and revision notes
- `07_Life_Arcs` era notes and transitions
- `08_Knowledge` topic notes not tied to one project
- `09_Assets` attachments, exports, screenshots
- `10_Dashboards` home and review notes
- `11_Templates` standard note templates
- `12_Audit` audits, stress tests, and review results
- `13_Pods` context bundles for output modes
- `14_Lenses` analytical frames
- `15_Agent_Prompts` reusable prompts for Claude Code or other agents
- `exports` packet-based exports for external AI analysis
- `scripts` helper scripts

## Design stance

Think in markdown.
Translate only on output.

Internal notes can stay closer to live reasoning.
Outbound work can be cleaned later.

## Current state

This is a real starter vault. It has actual content, not just empty scaffolding.
It is still meant to be lived in and revised.
The stronger it gets, the more it should reflect repeated use, not added complexity.

## Final review status

This v4 package is the post-audit build.

The earlier open policy questions are now settled:
- truly hidden material lives outside the main vault
- the main vault stores only shadow references and influence summaries
- executive default posture is restrained-authoritative with a Marcus Aurelius style archetype

This package is still a starter operating system, not a fake pre-lived archive.
What changed in v4 is that the policy spine is now closed, the audit is fresher, and the next steps are more executable.
